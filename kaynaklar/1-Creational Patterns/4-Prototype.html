Prototype Design Pattern

Spring Boot 3.5 REST API Implementation Guide

Pattern Tanımı ve Amacı

Prototype (Prototip), mevcut bir nesneyi kopyalayarak (cloning) yeni nesneler oluşturmanızı sağlayan bir creational (yaratımsal) tasarım desenidir. Bu desen, bir nesnenin ilk oluşturulma maliyeti yüksek olduğunda veya karmaşık nesnelerin farklı varyasyonlarını hızla oluşturmanız gerektiğinde oldukça verimli bir yaklaşımdır.

Temel Prensip: "Bir nesne oluşturmak için, ilk nesneyi sıfırdan oluşturmak yerine, halihazırda var olan bir nesneyi kopyala."

Neden-Sonuç İlişkisi

Problem: Maliyetli ve Karmaşık Nesne Oluşturma

Sorun: Geleneksel new keyword'ü ile nesne oluşturmanın zorlukları:
  • Yüksek Maliyet: Nesne oluşturma, veritabanı bağlantıları, dosya okuma veya yoğun CPU işlemleri gibi pahalı işlemler gerektirebilir.
  • Kod Tekrarı: Aynı veya benzer konfigürasyonlara sahip nesneleri oluşturmak için kod tekrarı yaşanır.
  • Sınıf Patlaması: Her küçük varyasyon için yeni bir alt sınıf (subclass) oluşturmak zorunda kalmak, sınıf hiyerarşisinin şişmesine neden olabilir.
  • Sıkı Bağlılık: İstemci kodu, somut (concrete) nesne sınıflarına bağımlı hale gelir, bu da esnekliği azaltır.
// Anti-pattern: Object creation with high overhead
public class ServerConnection {
    public ServerConnection(String config) {
        // Simulating a costly initialization process
        try {
            Thread.sleep(2000); 
            System.out.println("Connection to server established with config: " + config);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}

// A new instance is created every time, which is inefficient.
public class Client {
    public void performTask() {
        ServerConnection connection1 = new ServerConnection("PROD_CONFIG");
        ServerConnection connection2 = new ServerConnection("PROD_CONFIG"); // Repetitive costly operation
    }
}

Çözüm: Prototype ile Kopyalama

Sonuç: Prototype kullanımı ile:
  • Verimli Nesne Oluşturma: Ağır maliyetli nesne oluşturma işlemlerini tekrarlamak yerine, mevcut nesneler kopyalanır.
  • Dinamik Yapılandırma: Çalışma zamanında (runtime) nesneler klonlanabilir ve yeni konfigürasyonlarla anında güncellenebilir.
  • Genişletilebilirlik: Yeni bir nesne türü eklemek için alt sınıflar oluşturmaya gerek kalmaz. Sadece bir prototip nesnesi oluşturup onu klonlamak yeterlidir.
  • Gevşek Bağlılık: İstemci kodu, somut sınıfların karmaşık yapısını bilmeden sadece bir prototip nesnesiyle etkileşime girer.

Spring Boot 3.5 Implementation

Prototype Interface ve Concrete Sınıfı

// Step 1: Prototype Interface
public interface Document extends Cloneable {
    Document clone();
    void loadData();
    String getContent();
}

// Step 2: Concrete Prototype Class
@Component
public class ReportDocument implements Document {
    
    private String title;
    private String content;

    public ReportDocument() {
        // Public no-arg constructor is required for Spring
    }

    @Override
    public void loadData() {
        // Simulating a costly data loading operation
        this.title = "Financial Report " + System.currentTimeMillis();
        this.content = "Data from external source loaded.";
        System.out.println("Data loaded for: " + title);
    }
    
    @Override
    public Document clone() {
        try {
            // Shallow copy is enough for primitives/immutables
            return (Document) super.clone(); 
        } catch (CloneNotSupportedException e) {
            throw new IllegalStateException(e);
        }
    }
    
    // Getters and Setters...
}

Prototype Kayıt Mekanizması (Registry)

// Step 3: Prototype Registry/Cache
@Component
public class DocumentRegistry {
    
    private final Map<String, Document> prototypes = new HashMap<>();

    @PostConstruct
    public void init() {
        ReportDocument report = new ReportDocument();
        report.loadData(); // Costly initialization happens only once
        prototypes.put("financialReport", report);
    }

    public Document getDocumentPrototype(String type) {
        Document prototype = prototypes.get(type);
        if (prototype == null) {
            throw new IllegalArgumentException("Unknown document type: " + type);
        }
        return prototype;
    }
}

Service Layer: Klondalama İşlemi

// Step 4: Service Layer to use the prototype
@Service
public class DocumentService {
    
    private final DocumentRegistry registry;

    public DocumentService(DocumentRegistry registry) {
        this.registry = registry;
    }
    
    public String createAndCustomizeDocument(String documentType, String customTitle) {
        // Get the prototype from the registry (no costly creation here)
        Document prototype = registry.getDocumentPrototype(documentType);
        
        // Clone the prototype to create a new instance
        Document newDocument = prototype.clone();
        
        // Customize the new object without affecting the prototype
        ((ReportDocument) newDocument).setTitle(customTitle);
        
        return String.format("Document cloned with title: '%s' and content: '%s'",
                ((ReportDocument) newDocument).getTitle(), newDocument.getContent());
    }
}

REST Controller: Kullanım Örneği

// Step 5: REST Controller to expose the service
@RestController
@RequestMapping("/api/v1/documents")
public class DocumentController {
    
    private final DocumentService documentService;

    public DocumentController(DocumentService documentService) {
        this.documentService = documentService;
    }
    
    @GetMapping("/create")
    public ResponseEntity<String> createDocument(
            @RequestParam String type,
            @RequestParam String title) {
        
        try {
            String result = documentService.createAndCustomizeDocument(type, title);
            return ResponseEntity.ok(result);
        } catch (IllegalArgumentException e) {
            return ResponseEntity.badRequest().body(e.getMessage());
        }
    }
}

Sağladığı Faydalar

🚀 Performans Artışı
Nesne oluşturma maliyetini ortadan kaldırır. Özellikle karmaşık nesnelerin yaratılması için harcanan zamanı ve kaynakları önemli ölçüde azaltır.
🧬 Esnek ve Dinamik Yapılandırma
Çalışma zamanında farklı prototiplerden yeni nesneler oluşturarak, uygulamanın dinamik olarak farklı durumlara uyum sağlamasına olanak tanır.
✅ Kod Tekrarını Önleme
Benzer nesnelerin her seferinde sıfırdan oluşturulması yerine, bir şablon üzerinden klonlanması ile kod tekrarı (duplication) engellenir.
🚫 Alt Sınıf Patlamasını Engelleme
Her küçük fark için yeni bir alt sınıf oluşturmak yerine, ana prototip nesnesini klonlayarak ve klon üzerinde değişiklikler yaparak bu sorunun önüne geçilir.

Alternativeler ve Karşılaştırma

Pattern Use Case Prototype ile Farkı
Factory Method Hangi nesnenin oluşturulacağına alt sınıfların karar vermesi Doğrudan nesne yaratır. Prototipin aksine, mevcut bir nesneyi kopyalama mantığı yoktur.
Builder Karmaşık nesnelerin adım adım oluşturulması Sıfırdan bir nesne inşa ederken, Prototype mevcut bir nesneyi kopyalar. Birlikte de kullanılabilirler.
Singleton Uygulama genelinde tek bir nesne Her zaman aynı nesne instance'ını döndürürken, Prototype her çağrıda yeni bir kopya oluşturur.

✅ Prototype Avantajları

  • Verimlilik: Maliyetli nesne oluşturma işlemlerinden kaçınır.
  • Esneklik: Client kod, somut sınıflara bağımlı kalmaz.
  • Modülerlik: Nesne oluşturma ve klonlama mantığı, iş mantığından ayrılır.

⚠️ Dikkat Edilmesi Gerekenler

  • Shallow vs. Deep Copy: Nesne referansları varsa, derin kopyalama (deep copy) yapmak karmaşıklığı artırır.
  • Ekstra Kod: Basit nesneler için gereksiz yere karmaşıklık getirebilir.
  • Değiştirilebilirlik (Mutability): Kopyalanan nesnenin değiştirilmesi, kaynak nesneyi etkilememelidir; bu, dikkatli bir implementasyon gerektirir.

Kullanım Alanları

Prototype deseni, özellikle aşağıdaki durumlarda tercih edilir:

  • Objeleri sıfırdan oluşturmak, I/O veya hesaplama açısından maliyetli olduğunda.
  • Sisteminizde benzer özelliklere sahip birçok nesnenin dinamik olarak oluşturulması gerektiğinde.
  • Alt sınıf sayısını minimumda tutmak istediğinizde.
  • Bir oyunda aynı türden (örn. asker) binlerce nesne yaratılması gerektiğinde.
